# AI Chatbox — WordPress Plugin

A self-hosted, AI-powered customer support chat widget for WordPress. It answers visitor questions in a brand's own voice, grounded in documents the site owner uploads — no external SaaS chat platform, no per-seat pricing, just an Anthropic Claude API key.

Built by **Harry (Hai Ngoc Nguyen)**.

## What it does

- Embeds a floating chat widget on the front end, styled to a configurable accent color and position.
- Answers are generated by **Claude** (`claude-haiku-4-5`), grounded in a **free, keyword-searchable knowledge base** built from documents the admin uploads (`.txt`, `.pdf`) — no paid embeddings API, no vector database. Text is chunked and indexed with MySQL `FULLTEXT` search (`AICB_Knowledge_Base`), and the top-matching chunks are injected into the Claude system prompt as context.
- Supports **multiple chatbot profiles** on one site: each profile is a distinct persona (name, avatar, system prompt, welcome message, accent color, quick replies) scoped to a URL path prefix, with one profile marked as the site-wide default. This lets one install run, e.g., a general support bot plus a per-department or per-landing-page bot.
- All admin management (API key, upload limits, profiles, document upload/listing/deletion) happens from a single **Settings → AI Chatbox** page, driven by `wp_ajax` endpoints (`AICB_Ajax_Handler`) and rendered client-side (`admin/assets/admin.js`).

## Architecture

**Request flow** (`AICB_Ajax_Handler::handle_send_message`):
1. Resolve which profile owns the current page (`AICB_Profile_Resolver`, matched by URL path prefix, falling back to the default profile).
2. Search the knowledge base for chunks relevant to the visitor's message (`AICB_Knowledge_Base::search_chunks`, MySQL `FULLTEXT` `NATURAL LANGUAGE MODE`).
3. Build a Claude request: the profile's `system_prompt` + matched context chunks + conversation history + the new message (`AICB_Claude_Client::build_request_body`).
4. Call the Anthropic Messages API and return the reply.

**Knowledge base ingestion** (`AICB_Ajax_Handler::handle_upload_document`): an uploaded `.txt`/`.pdf` is parsed to plain text (`AICB_Document_Parser`, using `smalot/pdfparser` for PDFs), split into overlapping chunks (`AICB_Knowledge_Base::chunk_text`, 700 chars with 100-char overlap), and stored per-profile in the `chunks` table for full-text search.

**Data model** (`AICB_DB_Installer`, three tables created on activation via `dbDelta`):
- `wp_aicb_profiles` — one row per chatbot persona
- `wp_aicb_documents` — uploaded source files, scoped to a profile
- `wp_aicb_chunks` — chunked, full-text-indexed document content, scoped to a profile and document

**Key classes** (`ai-chatbox/includes/`):

| Class | Responsibility |
|---|---|
| `AICB_DB_Installer` | Creates/migrates the three DB tables on plugin activation |
| `AICB_Admin_Settings` | Global settings (API key, upload limits) registration and rendering |
| `AICB_Profiles` | Profile defaults, sanitization, path-prefix conflict checks |
| `AICB_Profile_Resolver` | Picks the active profile for a given request path |
| `AICB_Profile_Migrator` | One-time migration of pre-multi-profile installs into the new schema |
| `AICB_Knowledge_Base` | Text chunking + full-text search over uploaded documents |
| `AICB_Document_Parser` | Text extraction from `.txt`/`.pdf` uploads |
| `AICB_Claude_Client` | Builds and sends the Claude Messages API request |
| `AICB_Ajax_Handler` | All `wp_ajax_*` endpoints tying the above together |

## Installation

**Requirements:** WordPress 6.0+, PHP 7.4+, a MySQL/MariaDB version supporting `FULLTEXT` indexes on InnoDB, an [Anthropic API key](https://console.anthropic.com/).

1. Copy `ai-chatbox/` into `wp-content/plugins/` on your WordPress install (or upload one of the packaged `ai-chatbox-ver2*.zip` files via **Plugins → Add New → Upload Plugin**).
2. Install PHP dependencies if running from source rather than a packaged zip:
   ```bash
   cd ai-chatbox
   composer install
   ```
3. Activate the plugin — this creates the three database tables.
4. Go to **Settings → AI Chatbox** and set your Anthropic API key, upload limits, and at least one chatbot profile (mark it as Default).
5. Upload knowledge-base documents (`.txt`/`.pdf`) under a profile — the widget will ground its answers in them.
